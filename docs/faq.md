# FAQ

## What Java version is required?

Parsnip requires Java 11 or higher.

## What Jackson version does Parsnip use?

Parsnip uses Jackson 2.x. You need to register the Parsnip module with your `ObjectMapper`:

```java
ObjectMapper mapper = new YAMLMapper().registerModule(parsnipModule());
```

## Can I use JSON instead of YAML?

Yes. Since JSON is valid YAML, you can use either format with the same mapper. You can also use
`new ObjectMapper().registerModule(parsnipModule())` for pure JSON.

## How do I reference nested fields?

Parsnip supports [JSON Pointer notation](https://tools.ietf.org/html/rfc6901) for referencing nested fields,
e.g. `/messageBody/sensorId` to reference the `sensorId` field inside a `messageBody` object.

## How do I combine multiple filters?

Use `And`, `Or`, and `Not` to combine filters logically:

```yaml
And:
  - /field1: value1
  - Or:
      - /field2: value2
      - /field3: value3
```

## Are type comparisons strict?

By default, Parsnip uses *flexible* type comparisons, so the integer `10`, the float `10.0`, and the string `"10"`
are all considered equal. This can be helpful when working with data from different sources that may represent the
same value differently.
